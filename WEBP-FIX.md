# Решение проблемы с изображениями в старых браузерах

## Проблема
Все изображения в формате **.webp**, который не поддерживается в:
- Internet Explorer (все версии)
- Safari < 14
- Firefox < 65
- Chrome < 23
- Старые мобильные браузеры

## Решение 1: Конвертация в JPG (рекомендуется)

### Шаг 1: Установите Node.js зависимости
```bash
npm install sharp
```

### Шаг 2: Запустите конвертер
```bash
node convert-webp-to-jpg.js
```

Это создаст папку `img-jpg/` со всеми изображениями в формате JPG.

### Шаг 3: Обновите menu-final.json
Замените все пути к изображениям:
- Было: `"image": "img/0000-f0530b993edeb20e72e555caddc7fbde.webp"`
- Стало: `"image": "img-jpg/0000-f0530b993edeb20e72e555caddc7fbde.jpg"`

## Решение 2: Автоматический fallback (уже реализовано)

В файл `js/app-delivery.js` добавлена поддержка тега `<picture>`:
- Современные браузеры получают .webp
- Старые браузеры пытаются загрузить .jpg версию
- Если .jpg нет - показывается заглушка

**НО:** Для полной поддержки нужно сконвертировать файлы в JPG (см. Решение 1)

## Решение 3: Онлайн-конвертеры

Если не хотите устанавливать Node.js:
1. https://cloudconvert.com/webp-to-jpg
2. https://convertio.co/ru/webp-jpg/
3. https://www.iloveimg.com/ru/webp-v-jpg

Конвертируйте все 255 файлов и сохраните в папку `img-jpg/`

## Проверка работы

Откройте сайт в старом браузере и проверьте:
- ✅ Все изображения загружаются
- ✅ Нет битых картинок
- ✅ Сайт работает корректно

## Примечание

Тег `<picture>` уже добавлен в `js/app-delivery.js` (строка 934-940).
Осталось только сконвертировать файлы в JPG.
